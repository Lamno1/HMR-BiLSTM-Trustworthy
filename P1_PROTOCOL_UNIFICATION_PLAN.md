## 1. Root Cause Analysis

### 1.1 Three Separate Regimes Currently in Use

| Source | Model checkpoint | Test data | F1 metric | F1 value |
|--------|-----------------|-----------|-----------|----------|
| `report_results.py` | `best_rlstm.pt` (intra) | `data/processed/test.npz` (intra) | 5-class macro (includes Q) | 0.5644 |
| `evaluate_fgsm.py` / `evaluate_pgd.py` | `inter_best_rlstm.pt` (inter) | `splits/inter_test.npz` (inter) | 4-class macro `labels=[0,1,2,3]` | 0.8425 |
| `run_ablation.py` / `run_baselines.py` | `inter_best_rlstm.pt` (inter) | `splits/inter_test.npz` (inter) | 4-class macro `labels=[0,1,2,3]` | 0.8921 |

### 1.2 Why `table5_consolidated.csv` Contains a Physically Impossible Value

`generate_results_tables.py` assembles `table5_consolidated.csv` from multiple source CSVs:

- **`CLEAN["HMR-BiLSTM"]["f1"]`** ← `results/logs/baseline_results.json["hmr_bilstm"]["f1_macro"]`
  ← written by `report_results.py:309` → **5-class macro F1 = 0.5644** (intra model, intra data, wrong metric)

- **`FGSM["HMR-BiLSTM"]["adv_f1"]`** ← `results/tables/fgsm_baseline_summary.csv`
  ← written by `evaluate_fgsm.py:101` → **4-class macro F1 = 0.8425** (inter model, inter data, correct metric)

Because the FGSM column uses a **different metric definition** (4-class) than the Clean column (5-class),
FGSM-F1 (0.84) > Clean-F1 (0.56), which is physically impossible.

The PGD column has the same problem: PGD-F1 (0.8391) > Clean-F1 (0.5644).

## 2. Canonical Evaluation Protocol

### 2.1 Data Splits

| Split | Canonical path | Patients | Notes |
|-------|---------------|----------|-------|
| Train | `data/processed/splits/inter_train.npz` | DS1 minus val patients | Generated by `validation/preprocess_aami.py` |
| Val   | `data/processed/splits/inter_val.npz`   | DS1 val subset: 122, 124, 205, 223 | |
| Test  | `data/processed/splits/inter_test.npz`  | DS2 (all 24 patients) | Never used in training |

The paths above are already defined as constants in `configs/paths.py`:
```python
INTER_TRAIN = "data/processed/splits/inter_train.npz"
INTER_VAL   = "data/processed/splits/inter_val.npz"
INTER_TEST  = "data/processed/splits/inter_test.npz"
```

**Intra-patient splits** (`data/processed/train.npz`, `val.npz`, `test.npz`) must not be used for
any result that is reported in the paper.

### 2.2 Model Checkpoint

**Canonical checkpoint:** `results/checkpoints/inter_best_rlstm.pt`  
Path constant: `configs/paths.py:RLSTM_CKPT`

The old intra-patient checkpoint `results/checkpoints/best_rlstm.pt` must not be used for any
reported metric. It exists as an artifact of `train.py` and is strictly for legacy comparison.

### 2.3 Macro F1 Standard (AAMI EC57)

**All macro F1 computations must use `labels=[0,1,2,3]`** (N, S, V, F).

Class Q (label=4, paced/unknown beats) is excluded from AAMI macro F1 because:
- AAMI EC57 defines a separate evaluation track for pacemaker detection
- Q prevalence in MIT-BIH is < 2 %, so its effect on macro F1 is disproportionate
- Mixing 4-class and 5-class F1 across columns creates incomparable values

Correct call signature:
```python
f1_score(y_true, y_pred, labels=[0, 1, 2, 3], average="macro", zero_division=0)
```

Per-class reports should still include class Q; only the headline macro F1 must use 4 classes.

### 2.4 Robustness Evaluation Scope (Issue #12)

AutoAttack and CW must be evaluated on the **full inter-patient test set**, not a pre-filtered
subset. The current `outputs/v1.0_FINAL/robustness/autoattack_results.json` was computed on
n=200 samples selected because the clean model predicted them correctly, giving `clean_accuracy=1.0`
as the baseline — this makes the F1-drop figures incomparable to FGSM/PGD results (which run on
the full test set).

**Requirement:** Remove any pre-filtering step from `evaluate_autoattack.py` and from any CW
evaluation script before re-running.

### 2.5 Baseline Training Epoch Parity (Issue #41)

`run_baselines.py:587–611` trains LSTM and BiLSTM for 12 epochs.  
`train_inter_patient.py` trains HMR-BiLSTM for 45 epochs (from `configs/experiment_config.yaml`).

A 3.75× epoch budget difference confounds the comparison. **Baselines must be retrained with
45 epochs** to make the comparison fair.

## 4. Class Weight Source (Issue #6)

`run_baselines.py:562` loads `data/processed/class_weights.npy` generated by `preprocess.py`,
which computes weights from the **intra-patient training distribution**. Baselines train on
inter-patient splits, so the class weights are from the wrong distribution.

**Fix required in `run_baselines.py`:** Generate or load class weights from the inter-patient
training set (`inter_train.npz`) instead. A minimal change:

```python
# Instead of: np.load("data/processed/class_weights.npy")
y_inter_train = np.load("data/processed/splits/inter_train.npz")["y"]
counts = np.bincount(y_inter_train, minlength=5).astype(np.float64)
cw = counts.sum() / (5 * counts)
cw = np.clip(cw, 0.5, 50.0)  # same cap as preprocess.py after Fix 27
```
